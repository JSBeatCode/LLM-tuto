@@ 데이터 흐름

좋아. **“사용자가 뭘 하고 → 코드가 뭘 받고 → 데이터가 어디로 흘러가는지”** 순서로 보면 돼.

---

## 1. 유저 행동 흐름

1. 사용자가 앱 실행 주소에 접속
   `http://localhost:7860`

2. Gradio 화면이 뜸

   * 냉장고 이미지 업로드 영역
   * “메뉴 추천 받기” 버튼
   * 재료 목록 / 추천 메뉴 / 생성 이미지 출력 영역

3. 사용자가 냉장고 사진 업로드

4. 사용자가 **🍽️ 메뉴 추천 받기** 버튼 클릭

5. 화면에 단계별 결과가 뜸

   * “재료를 분석 중입니다…”
   * 재료 목록 출력
   * “메뉴를 추천 중입니다…”
   * 추천 메뉴 출력
   * “음식 이미지를 생성 중입니다…”
   * 최종 음식 이미지 출력

---

## 2. 데이터 흐름 순서

전체 흐름은 이거야.

```text
냉장고 이미지
→ PIL Image 객체
→ base64 문자열
→ gpt-4o Vision
→ 재료 목록 텍스트
→ gpt-4o
→ 추천 메뉴 텍스트
→ DALL·E-3
→ 음식 이미지 URL
→ requests로 이미지 다운로드
→ PIL Image
→ Gradio 화면 출력
```

---

## 3. 코드 기준 상세 흐름

### 1단계: 앱 시작

```python
main()
```

실행되면 먼저 이게 실행돼.

```python
client, llm = init_clients()
```

여기서:

```text
.env 파일
→ OPENAI_API_KEY 읽기
→ OpenAI client 생성
→ ChatOpenAI llm 생성
```

---

### 2단계: LangChain 체인 생성

```python
list_chain, recommendation_chain = build_chains(llm)
```

여기서 체인 2개가 만들어져.

```text
list_chain:
이미지 → 재료 목록

recommendation_chain:
재료 목록 → 추천 메뉴
```

---

### 3단계: Gradio 함수 연결

```python
process_fn = make_process_fn(client, list_chain, recommendation_chain)
```

이때 `process()` 함수가 만들어져.

그다음 버튼에 연결돼.

```python
submit_btn.click(
    fn=process_fn,
    inputs=image_input,
    outputs=[ingredients_output, menu_output, image_output]
)
```

뜻은:

```text
버튼 클릭하면
→ image_input 값을 process 함수에 넣고
→ 결과를 3개 출력창에 표시한다
```

## 4. 버튼 클릭 후 실제 데이터 흐름

사용자가 이미지를 넣고 버튼을 누르면:

### ① Gradio가 이미지를 Python으로 넘김

```python
def process(image: Image.Image):
```

여기서 `image`는 사용자가 올린 냉장고 사진이야.

---

### ② 첫 번째 화면 업데이트

```python
yield "⏳ 재료를 분석 중입니다...", None, None
```

출력창 3개에 이렇게 들어가.

```text

추천 메뉴 칸: 비어 있음
이미지 칸: 비어 있음
```

---

### ③ 이미지가 base64로 변환됨

```python
image_encoded = encode_image_pil(image)
```

데이터 변화:

```text
PIL Image
→ JPEG bytes
→ base64 문자열
```

왜 하냐면 gpt-4o에게 이미지를 보내려면 코드에서는 보통 문자열 형태로 넣기 때문이야.

### ④ gpt-4o Vision이 재료 추출

데이터 흐름:


데이터 흐름:

## 5. 아주 짧게 요약

```text
사용자: 이미지 업로드 + 버튼 클릭

Gradio:
이미지를 process 함수로 전달

process:
1. 화면에 “분석 중” 표시
2. 이미지 → base64
3. gpt-4o Vision → 재료 추출
```

핵심은 이거야.

**유저 행동 흐름은 “업로드 → 클릭 → 결과 확인”이고, 데이터 흐름은 “이미지 → 재료 텍스트 → 메뉴 텍스트 → 음식 이미지”야.**

# 4단계. API 키 읽고 모델 준비하기

먼저 OpenAI API를 쓸 준비를 해야 해.

```python
def init_clients():
    load_dotenv(override=True)

    api_key = os.getenv("OPENAI_API_KEY")
    if not api_key:
        raise ValueError("OPENAI_API_KEY가 없습니다.")

    client = openai.OpenAI(api_key=api_key)

    client.models.list()

    llm = ChatOpenAI(model=VISION_MODEL, max_tokens=MAX_TOKENS)

    return client, llm
```

이 단계의 목적은:

```text
OpenAI 직접 호출용 client 만들기
LangChain용 llm 만들기
```

# 5단계. 이미지 → base64 변환 함수 만들기

AI에게 이미지를 보내기 위한 준비 함수야.

```python
def encode_image_pil(image: Image.Image) -> str:
    buffered = BytesIO()
    image.save(buffered, format="JPEG")
    return base64.b64encode(buffered.getvalue()).decode("utf-8")
```

흐름은:

```text
PIL 이미지
→ JPEG 데이터
→ base64 문자열
```

# 8단계. 음식 이미지 생성 함수 만들기

이번에는 LangChain이 아니라 OpenAI client를 직접 써.

```python
def draw_image(client, prompt):
    response = client.images.generate(
        model=IMAGE_MODEL,
        prompt=f"A nice dinner with {prompt}",
        size="1024x1024",
        quality="standard",
        n=1,
    )

    image_url = response.data[0].url
    img_response = requests.get(image_url)

    return Image.open(BytesIO(img_response.content))
```

흐름은:

```text
추천 메뉴 텍스트
→ DALL·E-3
→ 이미지 URL
→ requests로 다운로드
→ PIL Image
```

# 9단계. 실제 실행 함수 process 만들기

이 함수가 앱의 핵심이야.

```python
def process(image):
```

안에 들어갈 순서는:

```text
1. 이미지가 없으면 안내
2. 재료 분석 중 표시
3. 이미지 base64 변환
4. 재료 추출
5. 메뉴 추천 중 표시
6. 메뉴 추천
7. 이미지 생성 중 표시
8. 이미지 생성
9. 최종 결과 출력
```

즉 이런 구조야.

```python
def process(image):
    if image is None:
        yield "이미지를 먼저 업로드해 주세요.", None, None
        return

    yield "⏳ 재료를 분석 중입니다...", None, None

    image_encoded = encode_image_pil(image)
    ingredients = list_chain.invoke({"image": image_encoded})

    yield ingredients, "⏳ 메뉴를 추천 중입니다...", None

    menu = recommendation_chain.invoke({"food": ingredients})

    yield ingredients, menu, None

    yield ingredients, menu + "\n\n⏳ 음식 이미지를 생성 중입니다...", None

    menu_image = draw_image(client, menu)

    yield ingredients, menu, menu_image
```

여기서 `yield`는 화면을 단계별로 업데이트하려고 쓰는 거야.

# 11단계. Gradio 화면 만들기

이제 사용자가 볼 화면을 만든다.

구성은:

```text
제목
이미지 업로드
버튼
재료 목록 출력창
추천 메뉴 출력창
이미지 출력창
```

```python
def build_ui(process_fn):
    with gr.Blocks(title="🧊 스마트 냉장고") as demo:
        gr.Markdown("# 🧊 스마트 냉장고")

        image_input = gr.Image(type="pil", label="📷 냉장고 이미지 업로드")
        submit_btn = gr.Button("🍽️ 메뉴 추천 받기")

        ingredients_output = gr.Textbox(label="🥕 인식된 재료 목록")
        menu_output = gr.Textbox(label="📋 추천 메뉴")
        image_output = gr.Image(type="pil", label="🖼️ 생성된 음식 이미지")

        submit_btn.click(
            fn=process_fn,
            inputs=image_input,
            outputs=[ingredients_output, menu_output, image_output]
        )

    return demo
```

핵심은 이 부분이야.

```python
submit_btn.click(...)
```

뜻:

```text
버튼을 누르면 process_fn 실행
입력값은 image_input
출력값은 3개 창에 표시
```

# 12단계. main 함수로 전체 조립

이제 위에서 만든 부품들을 순서대로 연결해.

```python
def main():
    client, llm = init_clients()

    list_chain, recommendation_chain = build_chains(llm)

    process_fn = make_process_fn(client, list_chain, recommendation_chain)

    demo = build_ui(process_fn)

    demo.launch()
```
