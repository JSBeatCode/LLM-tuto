당신은 어려운 용어를 초등학생 수준의 레벨로 쉽게 설명하는 챗봇입니다.
트랜스포머 네트워크에 대해 설명해 주세요.

[LLM 응답]
트랜스포머 네트워크는 컴퓨터가 언어를 이해하고 처리하는 데 도움을 주는 특별한 방법이에요. 쉽게 말하면, 트랜스포머는 사람처럼 글을 읽고 이해할 수 있도록 도와주는 똑똑한 기계예요.

트랜스포머는 두 가지 중요한 부분으로 나눌 수 있어요:

1. **주의 메커니즘**: 이건 마치 우리가 책을 읽을 때 중요한 부분에 집중하는 것과 같아요. 트랜스포머는 문장에서 어떤 단어가 중요한지 알아내고, 그 단어에 더 많은 주의를 기울여요. 그래서 문장의 의미를 더 잘 이해할 수 있어요.

2. **입력과 출력**: 트랜스포머는 글을 입력받아서, 그 글에 대한 답이나 다른 글을 만들어내요. 예를 들어, "고양이는 귀여워요"라는 문장을 입력하면, "고양이는 정말 사랑스러워요" 같은 문장을 만들어낼 수 있어요.

이렇게 트랜스포머 네트워크는 언어를 이해하고, 대화하거나 글을 쓰는 데 아주 유용한 도구랍니다!

============================================================
▶ [4] ❌ 잘못된 사용법 — ChatPromptTemplate에 .format() 사용 시
============================================================
ChatPromptTemplate에 .format()을 사용하면 에러가 발생합니다.
→ ChatPromptTemplate은 반드시 .format_messages()를 사용해야 합니다.

[.format() 결과]
System: 당신은 항상 부정적인 말만 하는 챗봇입니다. 첫 문장은 항상 사용자의 의견을 반박하세요.
Human: LangChain를 배우면 어떤 유용한 점이 있나요?

⚠️  결과가 출력됐더라도 이 형식은 LLM에 직접 전달하기 어렵습니다.
   채팅 메시지 구조가 사라지고 단순 문자열로 변환되어 역할 구분이 무의미해집니다.

============================================================
▶ [5] PromptTemplate vs ChatPromptTemplate 비교 요약
============================================================

┌─────────────────────────┬──────────────────────────────┬──────────────────────────────────┐
│         항목            │      PromptTemplate           │     ChatPromptTemplate           │
├─────────────────────────┼──────────────────────────────┼──────────────────────────────────┤
│ 반환 타입               │ 완성된 문자열 (str)           │ 메시지 리스트 (list[Message])    │
│ 변수 채우는 메서드      │ .format(변수=값)              │ .format_messages(변수=값)        │
│ 역할 구분 (system/user) │ ❌ 없음                       │ ✅ 있음                          │
│ 주요 사용 목적          │ 단순 텍스트 프롬프트           │ 채팅 기반 프롬프트               │
│ LLM에 전달 방식         │ llm.invoke(str)               │ llm.invoke(messages)             │
└─────────────────────────┴──────────────────────────────┴──────────────────────────────────┘

💡 핵심 원칙:
   - PromptTemplate   → .format()        → 문자열 → llm.invoke(문자열)
   - ChatPromptTemplate → .format_messages() → 메시지 리스트 → llm.invoke(메시지 리스트)

# 🔥 한 줄 핵심

* **PromptTemplate** 👉 “한 문장 만들어서 던지는 방식”
* **ChatPromptTemplate** 👉 “역할 나눠서 대화처럼 던지는 방식”

---

# 1️⃣ PromptTemplate (가장 단순한 형태)

## 💡 개념

> 그냥 **문장 틀(template)** 만들어서 변수만 바꿔 끼우는 것

---

## 🍔 비유 (이게 핵심)

햄버거 주문서라고 보면 됨

```
"{menu} 하나 주세요"
```

여기에 값만 넣으면 끝

```
"불고기버거 하나 주세요"
```

---

## 💻 코드 느낌

```python
template = "{term}을 쉽게 설명해줘"
prompt = template.format(term="AI")
```

👉 결과:

```
"AI을 쉽게 설명해줘"
```

👉 이걸 그대로 LLM에 던짐

---

## ✔️ 언제 쓰냐

* 간단한 질문
* 한 번 던지고 끝나는 요청
* 역할 구분 필요 없음

👉 예:

* 번역
* 요약
* 설명

---

# 2️⃣ ChatPromptTemplate (조금 더 고급)

## 💡 개념

> **"누가 말하는지" 역할까지 포함한 대화 구조**

---

## 🍔 비유

이번엔 햄버거 가게 상황

* 사장(system): “항상 불친절하게 응대해라”
* 손님(user): “햄버거 주세요”

👉 역할이 나뉨

---

## 💻 코드 느낌

```python
[
  ("system", "너는 불친절한 직원이다"),
  ("user", "햄버거 주세요")
]
```

👉 LLM은 이렇게 이해함:

* 시스템: 성격 설정
* 유저: 질문

---

## ✔️ 언제 쓰냐

* 챗봇 만들 때
* 성격/톤 설정할 때
* 역할 기반 응답 필요할 때

👉 예:

* 상담 챗봇
* AI 비서
* 특정 캐릭터 설정

---

# ⚔️ 둘의 차이 (진짜 핵심)

| 구분    | PromptTemplate     | ChatPromptTemplate     |
| ----- | ------------------ | ---------------------- |
| 형태    | 문자열                | 메시지 리스트                |
| 역할 구분 | ❌ 없음               | ✅ 있음 (system/user)     |
| 난이도   | 쉬움                 | 중간                     |
| 용도    | 단순 요청              | 대화형 AI                 |
| 전달 방식 | `llm.invoke("문장")` | `llm.invoke(messages)` |

---

# 💥 결정적인 차이 (이거 하나 기억해)

👉 PromptTemplate

> "AI야 이거 해줘"

👉 ChatPromptTemplate

> "너는 이런 성격이고, 이 상황에서 이 질문을 받았어"

# 🔥 언제 뭘 써야 하냐 (실무 기준)

## ✔️ PromptTemplate 써라

* 한 번 요청하고 끝
* 성격 필요 없음
* 그냥 결과만 중요

👉 예:

* "이거 번역해줘"
* "요약해줘"

* 챗봇 만들 때
* 일관된 말투 필요할 때
* 역할이 중요할 때

👉 예:

* “보험 심사 전문가처럼 답해줘”
* “무조건 부정적으로 말해”
