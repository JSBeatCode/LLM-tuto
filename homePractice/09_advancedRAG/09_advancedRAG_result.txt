💬 Answer  : EXAONE 언어 모델은 LG AI Research에서 개발한 최초의 오픈 모델로, 7.8억 개의 파라미터를 가지고 있으며, 영어와 한국어 모두에서 경쟁력 있는 성능을 보여줍니다. 이 모델은 특히 한국어 작업에서 뛰어난 성능을 발휘하며, 복잡한 추론 능력도 갖추고 있습니다. EXAONE은 전문가 수준의 AI 기능을 일반 대중이 접근할 수 있도록 민주화하는 것을 목표로 하고 있으며, 비상업적 및 연구 목적으로 공개되어 있습니다. 또한, EXAONE은 다양한 공공 및 내부 벤치마크에서의 평가를 통해 강력한 지침 준수 능력을 입증하였고, 실제 사용 사례에서의 성능이 우수하다는 점에서 다른 모델들과 차별화됩니다.
────────────────────────────────────────────────────────────

📌 Question: Phi-3 언어 모델은 어떤 데이터로 학습했나요?
💬 Answer  : Phi-3 언어 모델은 주로 두 단계로 구성된 데이터로 학습했습니다. 첫 번째 단계인 Phase-1에서는 일반 지식과 언어 이해를 가르치기 위해 웹 소스에서 수집한 데이터를 사용했습니다. 두 번째 단계인 Phase-2에서는 더욱 철저하게 필터링된 웹 데이터와 합성 데이터를 결합하여 모델의 논리적 추론 능력과 다양한 전문 기술을 향상시키는 데 중점을 두었습니다. 이러한 데이터 최적화 접근 방식은 작은 모델에서도 높은 성능을 달성할 수 있도록 설계되었습니다.

📌 Question: Qwen 2의 다국어 성능은 어떻게 나타났나요?
💬 Answer  : Qwen2는 약 30개 언어에서 강력한 다국어 능력을 보여주며, 다양한 벤치마크에서 경쟁력 있는 성능을 발휘합니다. 특히, Qwen2-72B 모델은 MMLU에서 84.2, GSM8K에서 89.5와 같은 주목할 만한 점수를 기록했습니다. 이러한 성능은 Qwen2의 다국어 처리 능력과 전반적인 언어 이해 및 생성 능력을 강조합니다.
────────────────────────────────────────────────────────────

📌 Question: Gemma의 스몰 모델은 어떻게 학습했나요?
💬 Answer  : Gemma의 스몰 모델은 지식 증류(knowledge distillation) 기법을 사용하여 학습되었습니다. 이 방법은 큰 모델을 교사로 삼아, 작은 모델이 각 토큰의 확률 분포를 학습하도록 하여 더 풍부한 기울기를 제공함으로써 훈련 시간을 단축시키는 방식입니다. 특히, 2B 및 9B 모델은 다음 토큰 예측 대신 지식 증류를 통해 훈련되었으며, 이는 모델의 성능을 크게 향상시키는 데 기여했습니다.

# 먼저 큰 그림

RAG는 기본적으로 이렇게 동작해.

```text
사용자 질문
   ↓
관련 문서 검색
   ↓
찾은 문서를 GPT에게 전달
   ↓
GPT가 답변
```

핵심은 사실 GPT보다:

```text
"관련 문서를 얼마나 잘 찾느냐"
```

야.

STEP2~5는 전부 검색 품질을 개선하는 단계들이야.

---

# STEP 2 = 기본 검색

가장 단순한 형태야.

## 동작 방식

```text
질문 1개
→ 의미 기반 검색 1번
→ 관련 문서 5개 찾기
→ GPT 답변
```

코드상으로는 이 부분.

```python
retriever = db.as_retriever(search_kwargs={"k": 5})
```

즉:

```text
"질문과 의미가 비슷한 문서"
```

를 찾는 거야.

---

## 비유

도서관 사서에게:

```text
"Phi-3가 어떻게 학습됐어?"
```

라고 물어보는 느낌.

사서가:

```text
"음… 이 질문과 비슷한 책 내용을 찾아볼게"
```

하고 찾아주는 거야.

---

## 문제점

질문 표현이 애매하면 검색 실패 가능성이 있어.

예:

```text
"Phi-3 학습 데이터?"
```

라고 했는데 논문에는:

```text
training corpus
pretraining dataset
synthetic data
```

라고 적혀있으면 못 찾을 수도 있어.

---

# STEP 3 = 질문을 여러 개로 바꿔서 검색

여기서 처음으로 “검색 강화 기술”이 들어가.

코드 이름:

```python
MultiQueryRetriever
```



---

## 핵심 아이디어

질문 하나를:

```text
LLM이 여러 개의 질문으로 변환
```

해버려.

---

## 예시

원래 질문:

```text
Phi-3는 어떤 데이터로 학습했나요?
```

LLM이 내부적으로:

```text
What datasets trained Phi-3?
What pretraining corpus was used?
How was Phi-3 trained?
```

이렇게 여러 버전 생성.

그리고:

```text
검색을 여러 번 수행
```

함.

---

## 비유

STEP2가:

```text
사서 1명에게 질문 1번
```

이라면,

STEP3은:

```text
질문을 여러 방식으로 바꿔서
사서에게 여러 번 물어봄
```

이야.

---

## 장점

검색 누락 감소.

즉:

```text
"아 이런 표현도 있었네?"
```

를 더 잘 잡아냄.

---

## 단점

LLM을 추가로 사용함.

즉:

```text
비용 ↑
속도 ↓
```

---

# STEP 4 = 검색 엔진 2개를 섞음

여기서 진짜 중요한 개념 등장.

```text
의미 검색 + 키워드 검색
```

을 같이 사용해.

코드:

```python
EnsembleRetriever
```



---

# 왜 두 개를 섞어?

검색 방식마다 장단점이 달라.

---

## 1. Semantic Search (의미 검색)

STEP2에서 쓰던 방식.

```text
"뜻이 비슷한 문장"
```

을 찾음.

예:

```text
질문:
"다국어 성능"
```

논문:

```text
multilingual capability
```

뜻이 비슷하니까 찾을 수 있음.

---

## 2. BM25 (키워드 검색)

이번에 새로 추가됨.

이건:

```text
"단어 자체"
```

를 찾음.

예:

```text
Qwen2
```

라는 단어가 정확히 들어있는 문서를 잘 찾음.

---

# 비유

Semantic 검색:

```text
"뜻이 비슷한 책"
```

BM25 검색:

```text
"단어가 정확히 들어간 책"
```

---

# STEP4는?

둘을 합쳐버림.

```text
BM25 + Semantic
```

---

## 비유

STEP2:

```text
감으로 잘 찾는 사서 1명
```

STEP4:

```text
감 좋은 사서 + 키워드 전문 사서
둘이 같이 찾음
```

---

## 장점

검색 정확도가 많이 좋아짐.

특히:

```text
모델명
숫자
전문용어
```

찾을 때 강함.

---

# STEP 5 = 문서에 "설명 태그"를 미리 붙여놓음

여기가 제일 고급 기술이야.

이름:

```text
Contextual Retrieval
```



---

# STEP2~4의 근본 문제

RAG는 문서를 잘게 잘라(chunk) 저장해.

예:

```text
[청크1]
The model was trained...

[청크2]
Synthetic data was used...

[청크3]
Evaluation results showed...
```

문제는:

```text
청크만 보면 무슨 내용인지 애매함
```

이야.

---

# STEP5의 아이디어

LLM이 각 청크 앞에:

```text
"이 청크는 무슨 내용인지"
```

짧은 설명을 미리 붙여버림.

---

## 예시

원래:

```text
Synthetic data was used...
```

STEP5:

```text
This chunk explains Phi-3 training data strategy.

Synthetic data was used...
```

---

# 효과

검색기가 이해하기 쉬워짐.

즉:

```text
"이 청크가 무슨 역할인지"
```

미리 설명이 붙어있어서 검색 품질 향상.

---

# 비유

STEP2~4:

```text
책 페이지 조각만 보고 찾기
```

STEP5:

```text
각 페이지 조각 위에
"이 페이지는 이런 내용"
메모가 붙어 있음
```

---

# 왜 고급 기술이야?

모든 chunk마다 GPT 호출함.

즉:

```text
청크 300개
→ GPT 300번 호출
```

가능.

그래서:

```text
비용 ↑↑↑
시간 ↑↑↑
```

---

# 핵심 차이를 한 장으로 정리

| STEP  | 핵심 아이디어            | 쉽게 말하면          |
| ----- | ------------------ | --------------- |
| STEP2 | 기본 의미 검색           | 질문 1번 검색        |
| STEP3 | 질문 여러 개 생성         | 여러 방식으로 검색      |
| STEP4 | BM25 + Semantic 혼합 | 키워드 + 의미 둘 다 사용 |
| STEP5 | 청크에 설명 추가          | 문서에 미리 요약 태그 붙임 |

---

# 진짜 중요한 핵심

사실 STEP2~5는:

```text
GPT를 바꾸는 게 아니라
검색 품질을 개선하는 기술
```

이야.

LLM은 거의 동일해.

바뀌는 건:

```text
"GPT에게 어떤 문서를 전달하느냐"
```

야.
